MySQL之DQL

发表于 2019-11-12 | 分类于 MySQL

基础语法

基础查询

  1. 查询多个字段
    select 字段名1,... from 表名;
    • 查询所有字段
      select * from 表名;
  2. 去除重复的结果
    select distinct 字段名 from 表名;
  • 有多个字段名时，只有各行数据完全相同才去重
  1. 列求和
    select 列名1+列名2 from 表名;
  • 如果有null参与的运算，结果全为null
  • ifnull()函数
    select ifnull(含null的列,替换成的值) from 表名;

    条件查询

    where 条件
  1. 关系运算符：>, <, >=, <=, =, （两个不等于）!=, <>
  2. 与：&&, and, between…and…
  3. 或
  4. null值不能使用=（!=）判断

    模糊查询(like)

  • 占位符
    ‘_’表示单个字符
    ‘%’表示多个字符
  • where 字段 like ‘…’

    排序查询

  • order by 排序字段1 排序方式1， 排序字段2 排序方式2，......
    select * from stu order by math asc/desc;

asc : 升序

desc : 降序

如果不加这两个限定符，则默认用升序

  • 要对多个字段都排序，当前面的条件满足时，才会去判断第二条件
    select * from stu order by math asc,english desc;

    聚合函数

    将一列数据作为一个整体，进行纵向的计算

  • 聚合函数会排除null值

    • 解1：ifnull函数
      select count(ifnull(english,0)) from stu;
    • 解2：选择不含null的列计算
      count(*)

      count():计算个数
      max():最大值
      min():最小值
      sum():求和
      avg():平均数

分组查询

  1. 分组查询的字段：分组字段，聚合函数
    select sex,avg(math),count(id) from stu group by sex;
  2. where和having的区别
    1.where在分组前进行限定，如果不满足条件，则不参与分组；having在分组后进行限定，如果不满足条件，则不会被查询出来
    1. where后不可以跟聚合函数，having后可以进行聚合函数的判断

      分页查询

      limit 开始的所有，每页查询的条数;
      公式：开始的所引 = （当前页码 - 1） * 每页显示的条数
